# Hat Principle（帽子原则）

**Hat Principle（帽子原则）** 是 Toka 内存安全模型背后的核心概念。它不是要求开发者标注生命周期或手动管理内存，而是使用称为"帽子（hat）"的可视语法标记，在编译时追踪所有权和内存语义。

## 什么是帽子？

**帽子（hat）** 是附加在标识符上的一个标记符号（sigil），告诉程序员和编译器该值是如何存储、拥有和访问的。Toka 有四种帽子类型：

| 帽子 | 名称 | 含义 |
|------|------|------|
| `*` | 原始指针 | 底层指针，需要 `unsafe` / `alloc` |
| `^` | 独占指针 | 对堆分配资源的独占所有权（类似 Rust 的 `Box`） |
| `~` | 共享指针 | 基于引用计数的共享所有权 |
| `&` | 借用指针 | 对其他值的引用/借用 |

## 手柄 vs. 灵魂

Toka 最优雅的设计特性之一是**手柄（Handle）vs. 灵魂（Soul）** 的区分。它消除了显式解引用运算符的需求：

- **手柄（帽子）**：**带有**标记符号的标识符（`^p`, `*p`, `~p`）表示*指针容器本身*。
- **灵魂（数据）**：**不带**标记符号的标识符（`p`）表示*底层数据*。

当你想要读取或修改指针指向的值时，你直接在**灵魂**上操作——不需要解引用标记：

```toka
{{#include ../examples/ownership.tk:raw_ptr}}
```

这是对 C 语言的 `*p = 200` 或 Rust 的 `*p = 200` 的重大生活质量改进。Toka 的方法使指针密集型代码更加可读。

## PAL Checker 如何使用帽子

PAL（Pointer Analysis Layer，指针分析层）检查器在编译时分析帽子的使用情况，以强制执行借用检查器所提供的相同保证，但无需生命周期标注：

1. **移动后不可使用** — `^` 值在所有权转移后不能使用
2. **无悬垂引用** — 引用不能超过其引用对象的生命周期
3. **无双重释放** — 每个拥有的值恰好释放一次
4. **无数据竞争** — 可变访问是独占的；共享访问是只读的
